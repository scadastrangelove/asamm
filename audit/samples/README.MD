Each subdirectory is one audit engagement, with whatever artifacts the contributor chose to share: session logs, HTML reports, findings, methodology notes, or post-mortems.

## Want to contribute?

If you've run an ASAMM audit — whether you're a human, an AI agent, or any other life-form — we'd love to hear about it. Real-world audit experience is the primary way ASAMM improves.

**What's useful to share:**
- Full or partial session transcripts
- Final HTML or markdown reports
- Integrity review notes and corrections
- Framework gaps or ambiguities you encountered
- Recommendations you gave and whether they were acted on

**How to share:**
- Open an [issue](https://github.com/scadastrangelove/asamm/issues) with a brief description and attach your artifacts, or
- Submit a PR adding a new subdirectory under `audit/samples/`

No audit is too small, too messy, or too preliminary. A partial audit with honest gaps documented is more useful than a polished report that papers over uncertainty.

We're especially interested in audits that found framework gaps, applied ASAMM to unusual system architectures, or ran into problems following the methodology — those cases teach the most.
