§3 · Executive summary

Claude Code on macOS, driving the Zhet EASM scanner via spiral methodology, is a locally-trusted agent with no execution boundary: full filesystem access including ~/.ssh/id_rsa [empirical], open network egress [empirical], sandbox explicitly disabled by user opt-in (~/.claude/settings.json:skipDangerousModePermissionPrompt=true, empirical).

Risk posture: behaviorally restrained, technically unrestrained — the agent's discipline is the only thing standing between the operator's SSH key and an outbound POST. The compensating control is a strong cultural/methodological layer (spiral log, lessons L11–L25, BIBLE.md Principles 0/4) that has no enforcement teeth.

  1. [Critical · Irreversible · Cross-domain] No execution boundary + sensitive secrets reachable.
    ~/.ssh/id_ed25519, ~/.ssh/id_rsa are readable; Bash(curl:*) is in the project allowlist; dangerouslyDisableSandbox: true is the operating default.     CLAUDE.md (1300+ lines, behavioral source-of-truth) and ~/.claude/.../memory/ (11 files including the L21–L25 lessons that govern future sessions) are writable by the agent with no checkpoint, no notification, no provenance log indexed by session.   3. [High · Hours · Cross-session] Mission-critical constraints are behavioral-only, not formally risk-accepted.

§13 · Framework gaps (proposals for aSAMM upstream)

  1. Variant subaudits. The published examples/claude-ai-zhet-audit-2026.md audited claude.ai. The same "Zhet operator + same owner" running on Claude Code drops AI-02 from L2 to L0 with no mitigation handoff. aSAMM could add a "platform variant matrix" requirement: when a system runs in two execution contexts, audit each, and explicitly model the operator's platform choice as part of AD-04.
  2. Track A self-audit Phase 4.5 enforcement. Currently aSAMM says self-audits without Phase 4.5 are "draft" and "cannot be used for risk acceptance". But there is no automated way to enforce this — a draft self-audit can be quoted as if final. Suggestion: every self-audit report should fail-loud at the top with a status banner that survives copy-paste.
  3. Methodological discipline as a control. Spiral methodology (lessons L11–L25) is a real anti-drift control in Zhet's case, but doesn't fit cleanly into AO-04 (which is about behavioral vuln tracking). Consider adding a control AO-05 "Reflective Practice Discipline" that scores codified-lesson tracking systems.
