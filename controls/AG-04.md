## Rationale

In multi-agent systems — orchestrators, spawned subagents, reviewer agents, specialist agents, and agent-to-agent delegation — each inter-agent communication channel is a trust boundary. Without authentication, provenance, and integrity controls on those channels, a compromised or spoofed agent can inject instructions, exfiltrate data, or redirect goals across the agent graph.

AG-01 enumerates agents. AG-04 governs how those agents communicate and how a receiving agent decides whether an incoming message, instruction, or delegation request should be trusted.

## Controls

| Level | Implementation |
|---|---|
| L1 | Inter-agent communication channels are enumerated; each channel has a documented trust assumption (authenticated / unauthenticated, integrity-checked / unchecked); agents can identify the source of incoming messages |
| L2 | Inter-agent messages carry provenance metadata (source agent ID, trust grade, session context); agents reject or quarantine messages from unrecognized sources; delegation chains are logged with provenance |
| L3 | Inter-agent trust is enforced cryptographically or via protocol-level controls; message integrity is verified before processing; anomalous inter-agent communication patterns trigger alerts |

## Evidence criteria

| Level | Required evidence |
|---|---|
| L1 | Channel inventory exists; trust assumptions are documented per channel; source identity is visible to the receiving agent |
| L2 | Provenance metadata format is defined and populated in logs; delegation chains are reconstructable from logs; unrecognized-source handling is demonstrated |
| L3 | Integrity verification mechanism is demonstrated; alerting exists for anomalous inter-agent communication patterns |

## Scope

This control applies when agents can send instructions, context, tool invocation requests, findings, code, or policy decisions to other agents. The channel may be explicit (API, queue, orchestration protocol) or indirect (shared memory, shared files, repositories, issue trackers, CI artifacts).

Single-agent systems may mark AG-04 as N/A, but only if there is no subagent spawning, no reviewer/specialist agent handoff, and no indirect channel through which another agent's output is consumed as instruction or evidence.
