**AG-01 — Agent Registry and Autonomy Classification**

SAMM function: Governance | Threat coverage: C1, C3, W1 | Path: B

| Level | Implementation |
|---|---|
| L1 | Agents are enumerated with owner, tool set, and autonomy level documented in a maintained register |
| L2 | Autonomy levels are formally classified (e.g., supervised / semi-autonomous / autonomous); each classification defines its review requirements |
| L3 | Autonomy level changes require formal re-classification; classification is reviewed at each spiral turn and after any capability increase |

Evidence: agent registry exists and is current; each entry has a named owner; autonomy level is documented and matches deployed configuration.

*Trust grading in the agent registry.*

Autonomy classification alone is insufficient. An agent may be classified as semi-autonomous and still be an F-grade source — newly deployed, behaviorally untested, with no operational track record. The agent registry should record both autonomy level and trust rating (see Section 0.5). Autonomy expansion requires trust rating improvement, not only technical capability demonstration. An agent cannot hold higher autonomy than its trust rating supports.

## v0.2 extension — Dynamic Agent Graphs

*Added: ASAMM v0.2*

AG-01 as originally specified assumes a fixed agent registry. Systems where agents can spawn subagents (e.g., Claude Code Agent tool, multi-agent orchestrators) require additional handling:

**Spawning capability classification:**
If the registered agent can spawn subagents, the spawning capability itself must be classified as part of the parent's autonomy classification. Specifically:
- What is the maximum delegation depth? (can spawned agents themselves spawn further agents?)
- What tool surface does the spawned agent receive? (bounded by parent authorization, or full parent grant?)
- Does the spawning event appear in the action log?

**Default trust for spawned agents:**
Spawned agents start at **F6** trust regardless of the parent agent's trust rating. Trust rating is not inherited. Upgrade requires the same behavioral confirmation process as any new agent.

**L2 addition for dynamic graphs:**
> If agents can spawn subagents: spawning capability classified; spawning events logged; each spawned agent's tool surface is bounded by parent authorization (not the parent's full tool grant); parent autonomy classification includes maximum delegation depth.

**Evidence addition:**
> If subagent spawning is possible: spawning event log exists and is current; tool surface of spawned agents is demonstrably bounded; no spawned agent holds broader tool access than its parent's explicit delegation.
