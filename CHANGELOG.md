# Changelog

All notable changes to Agentic SAMM are documented here.

Format follows [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).
Versioning follows [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Control IDs (AG-01, AD-01, etc.) are stable identifiers. Changes to existing control IDs are breaking changes and increment the minor version.

## [v0.1.0-draft] — 2026

### Added

- Preface with authorship context and dual-audience framing
- Part 0 — Foundations: five agentic security axioms, four core concepts, threat taxonomy
- Agentic Threat Taxonomy v0.2: four primary threat classes (C1–C4), two weakness overlays (W1–W2), three ecosystem modifiers (E1–E3)
- Attack pattern examples: Context Injection, Tool Abuse, Autonomy Window Exploitation
- Part 1 — Migration Path: structured gap analysis for existing SAMM programs
- Inherited False Positives section: seven common false-confidence signals
- Part 2 — Greenfield Path: minimum secure baseline and priority-ordered controls
- Greenfield False Starts section: six premature-confidence failure modes
- Greenfield Readiness Assessment: eight outcome-based criteria
- Part 3 — Shared Control Reference: 17 controls across five SAMM function families
- L1/L2/L3 maturity levels with evidence criteria per control
- Framework mapping: NIST AI RMF, NCSC Secure AI Guidelines, MCP Security Best Practices
- Agentic Vulnerability Lifecycle: seven-stage process for behavioral vulnerabilities
- Six figures: spiral model, threat taxonomy, autonomy window, maturity levels, two paths, taxonomy-to-controls mapping
- Stable control IDs: AG-01..03, AD-01..04, AI-01..03, AV-01..03, AO-01..04

### Scope

Initial draft release. Scope is frozen at v0.1.0. Future minor versions will add controls or extend taxonomy. Patch versions will clarify wording, improve examples, and tighten evidence criteria.
